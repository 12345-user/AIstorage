# 自动知识库工作流

更新时间：2026-06-08

本工作流必须与 `06_Skills/知识库输入标准与指令识别规范.md` 保持一致。

## 总流程

1. 外部信息进入 `00_Inbox`。
2. Codex 读取标准识别区和结构化指令：`CMD`、`PROJECT`、`ACTION`、`OUTPUT`、`TARGET`、`PRIORITY`。
3. Codex 按五项规则检查：相关性、新鲜度、价值度、可输出性、关联性。
4. 按 P1-P5 或 X 共享区分流。
5. 原始资料进入 `01_Sources`，观点拆成 `02_Knowledge`，选题进入 `03_Topics`。
6. 可交付内容进入 `04_Outputs`。
7. 每日简报、每周复盘、月度总结进入 `05_Review`。

## 自动抓取流

| 来源 | 默认指令 | 初始保存 | 二次处理 |
| --- | --- | --- | --- |
| 行业新闻 / 热点 / RSS | `CMD:整理 | PROJECT:待判断 | ACTION:B+S | OUTPUT:每日简报` | `00_Inbox/自动抓取` | 重要内容转 `01_Sources/行业新闻` 或项目资料 |
| GitHub Trending / AI 工具 | `CMD:整理 | PROJECT:P4 | ACTION:B+S | OUTPUT:每日简报` | `00_Inbox/自动抓取` | P4 转 `10_Projects/04_AI前沿与Codex工具/01_新闻趋势` |
| 论文 / 报告 | `CMD:整理 | PROJECT:P2 | ACTION:B+A | OUTPUT:报告` | `00_Inbox/自动抓取` | P2 转 `01_Sources/论文报告` 或项目文献 |
| 网页文章 | `CMD:整理 | PROJECT:待判断 | ACTION:B+A/S` | `00_Inbox/手动剪藏` | 转 `01_Sources/深度文章` |
| 社媒链接 | `CMD:整理 | PROJECT:P1 | ACTION:B+S | OUTPUT:视频脚本` | `00_Inbox/手动剪藏` | 转 `01_Sources/社媒内容` 或 P1 热点素材 |
| 飞书 / 会议纪要 | `CMD:整理 | PROJECT:待判断 | ACTION:B+A | OUTPUT:周报` | `01_Sources/会议纪要` | 提炼行动项和知识卡 |
| Word / 旧报告 | `CMD:整理 | PROJECT:待判断 | ACTION:B+C` | `01_Sources/旧资料导入` | 转资料笔记或输出归档 |

## Codex 自动检查输出

每次整理新资料时，Codex 输出：

- 标准识别区：完整 / 缺字段。
- 分类结果：`PROJECT`、`ACTION`、`OUTPUT`。
- 推荐路径：`TARGET` 或推断路径。
- 一句话摘要。
- 关联笔记：能链接则链接，不能链接则写“暂无”。
- 下一步动作：继续研究、生成脚本、做知识卡、纳入周报等。

## 每日简报流

1. 从 `00_Inbox/自动抓取`、`00_Inbox/手动剪藏`、项目收件箱读取当天新增内容。
2. 只处理符合统一标准或可补齐标准识别区的内容。
3. 按项目分组输出：P1、P2、P3、P4、P5、X。
4. 列出值得进一步输出的选题。
5. 保存到 `05_Review/每日简报/YYYY-MM-DD.md`，使用 `_templates/每日简报模板.md`。
