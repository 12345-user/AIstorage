# Codex 自动检查规则

当 Codex 帮你整理资料时，默认按以下流程处理。

## 先做项目归属判断

- P1 客栈自媒体脚本与热点
- P2 生物信息研究生课程与实验报告
- P3 大学课程日常跟进
- P4 AI 前沿与 Codex 工具
- P5 其他杂事项目
- X 共享资料

## 五项判断

1. 相关性：是否服务于研究领域或当前输出。
2. 新鲜度：是否包含新趋势、新案例、新工具或新数据。
3. 价值度：是否值得保存、引用、研究或复盘。
4. 可输出性：是否能变成文章、报告、脚本、课程或周报。
5. 关联性：是否能连接已有笔记或主题。

## 推荐动作

| 判断 | 动作 | 保存位置 |
| --- | --- | --- |
| 属于具体项目 | P 进项目 | `10_Projects` 对应项目 |
| 可被多个项目复用 | X 进共享区 | `20_Shared` |
| 可形成选题 | S 进选题池 | `03_Topics/选题池` |
| 可沉淀观点 | A 做知识卡 | `02_Knowledge` |
| 适合留作引用 | B 存参考资料 | `01_Sources` |
| 价值一般但需保留 | C 归档 | 对应资料目录 |
| 低价值或重复 | D 忽略 | 不入库 |

## 输出格式

每次整理资料时，尽量给出：

- 分类结果
- 推荐保存位置
- 一句话摘要
- 可关联的已有笔记
- 下一步动作
