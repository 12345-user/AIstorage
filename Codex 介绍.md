# Codex 介绍

Codex 是 OpenAI 面向软件开发和知识工作的编码智能体。它可以阅读和理解代码库，协助编写、修改、审查、调试代码，也可以配合本地工具、浏览器、文档和自动化流程完成更长链路的任务。

## 它适合做什么

- 理解现有项目结构，定位相关文件和实现逻辑。
- 按需求修改代码，并尽量保持项目原有风格。
- 运行测试、构建或检查命令，验证改动是否可用。
- 做代码审查，找出潜在缺陷、回归风险和缺失测试。
- 构建前端页面、交互工具、脚本、报告和工程化辅助文档。
- 将重复流程沉淀为技能、插件、自动化或仓库规则。

## 工作方式

Codex 的核心不是只给一段答案，而是和用户共享一个工作区，边读项目边行动。一次典型协作通常包括：

1. 先观察仓库结构、已有约定和相关文件。
2. 根据任务选择最小必要改动。
3. 编辑文件或创建新文件。
4. 运行可用的验证命令。
5. 向用户说明改了什么、验证了什么、还有哪些风险。

这种方式适合真实项目中的渐进式开发，因为它会尽量尊重已有代码、文档和团队习惯。

## 常见使用场景

- 新功能开发：从需求说明到代码实现和测试补充。
- Bug 修复：复现问题、定位原因、提交小范围修复。
- 前端实现：根据描述、截图或设计稿搭建可交互页面。
- 代码库迁移：升级 API、替换旧依赖、整理兼容逻辑。
- 文档维护：编写 README、架构说明、使用指南和变更记录。
- 质量检查：代码审查、测试补全、安全和性能风险梳理。

## 和普通聊天机器人的区别

普通聊天更像是“回答问题”。Codex 更像是“进入项目一起做事”：它可以读取仓库文件、运行命令、编辑代码，并根据执行结果继续调整。对工程任务来说，这种闭环比单次建议更有价值。

## 使用建议

- 说明目标，而不只是说明某个文件要怎么改。
- 如果有约束，提前写清楚，比如技术栈、风格、不能改动的范围。
- 让 Codex 运行测试或检查命令，减少只看代码带来的误判。
- 对长期规则，可以考虑写入仓库文档或 Codex 可读取的指导文件。
