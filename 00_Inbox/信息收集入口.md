# 信息收集入口

所有暂未处理的信息先放在这里。本入口使用全 Vault 统一标准：`06_Skills/知识库输入标准与指令识别规范.md`。

## 最小输入格式

```text
标题：
链接 / 内容：
PROJECT：P1 / P2 / P3 / P4 / P5 / X / 待判断
CMD：整理 / 入库 / 知识卡 / 选题 / 脚本 / 文章 / 报告 / 周报 / 简报
```

## 完整输入格式

```text
CMD: 整理
PROJECT: P1 / P2 / P3 / P4 / P5 / X / 待判断
ACTION: P / X / S / A / B / C / D
OUTPUT: 文章 / 报告 / 视频脚本 / 视频摘要 / 课程 / 周报 / 每日简报 / 暂无
TARGET:
PRIORITY: 高 / 中 / 低
NOTE:
```

## 进入方式

- 自动抓取：放入 `00_Inbox/自动抓取`，使用 `_templates/自动抓取资料模板.md`。
- 手动剪藏：放入 `00_Inbox/手动剪藏`，使用 `_templates/资料笔记模板.md`。

## 处理检查

- 相关性：是否和研究领域、项目或输出目标有关？
- 新鲜度：是否足够新，是否包含新趋势、新案例、新工具或新数据？
- 价值度：是否值得保存、引用或研究？
- 可输出性：是否能转化为文章、报告、脚本、课程或周报？
- 关联性：是否能连接已有笔记、主题、资料或输出？

## 分流结果

- `P`：进入项目。
- `X`：进入共享区。
- `S`：进入选题池。
- `A`：做成知识卡。
- `B`：作为参考资料保存。
- `C`：归档。
- `D`：忽略。
