# 共享资料说明

`20_Shared` 用来存放多个项目都会复用的资料。所有共享资料使用统一标准：`06_Skills/知识库输入标准与指令识别规范.md`。

## 标准识别区

- 输入ID：shared-index
- 输入时间：2026-06-08
- 来源类型：手动记录
- 来源名称：共享资料说明
- 作者 / 机构：
- 原始日期：2026-06-08
- 原始链接：暂无
- 项目归属：X
- 内容类型：资料
- 目标输出：文章 / 报告 / 视频脚本 / 视频摘要 / 课程 / 周报 / 每日简报 / 暂无
- 处理指令：整理
- 分类动作：X
- 优先级：高
- 状态：已入库
- 标签：共享资料

## 默认指令

```text
CMD:整理 | PROJECT:X | ACTION:X/A/S/C | OUTPUT:暂无 | TARGET:20_Shared | PRIORITY:中
```

## 子目录

- `账号与渠道`：平台账号、对标账号、发布渠道、内容格式。
- `选题池`：跨项目可复用选题。
- `写作方法`：脚本、文章、报告、课程材料的写作方法。
- `AI工具`：AI 工具、Codex 工作流、提示词和自动化配置。
- `客栈素材库`：客栈基础介绍、图片说明、卖点、常见问题。

## 使用规则

- 只服务单个项目的资料，不放这里。
- 能被两个以上项目复用的资料，放这里。
- 项目内笔记可以链接这里的共享资料。
- 共享资料如果可转成知识卡，使用 `CMD:知识卡 | PROJECT:X | ACTION:A`。
