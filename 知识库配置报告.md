## 统一标准

所有新资料使用同一套字段：

```text
CMD:整理 | PROJECT:P1/P2/P3/P4/P5/X/待判断 | ACTION:P/X/S/A/B/C/D | OUTPUT:文章/报告/视频脚本/视频摘要/课程/周报/每日简报/暂无 | TARGET:保存路径 | PRIORITY:高/中/低
```

最小输入格式：

```text
标题：
链接 / 内容：
PROJECT：P1 / P2 / P3 / P4 / P5 / X / 待判断
CMD：整理 / 入库 / 知识卡 / 选题 / 脚本 / 文章 / 报告 / 周报 / 简报
```

所有模板必须包含：

- 标准识别区。
- 一句话摘要。
- Codex 五项检查。
- 推荐去向。

## 完整功能目标

完整功能指的是：

1. 自动抓取：GitHub、RSS、论文、行业新闻、热点资料能自动进入 `00_Inbox/自动抓取`。
2. 手动剪藏：网页、社媒、公众号、视频、灵感能快速进入 `00_Inbox/手动剪藏`。
3. 自动分流：Codex 能按 P1-P5/X 和 P/X/S/A/B/C/D 统一标准分流。
4. 自动沉淀：重要资料能转成 `01_Sources`、`02_Knowledge`、`03_Topics`。
5. 自动输出：能生成每日简报、周报、文章、报告、视频脚本、视频摘要。
6. 附件管理：图片、PDF、表格、视频素材能进入 `99_Attachments` 并被笔记引用。
7. 版本同步：Vault 能稳定提交并推送到 GitHub。

## 分步配置路线

### 第 1 步：确认 GitHub 同步

目标：让 Vault 可以稳定备份和版本管理。

你需要确认：

```text
GitHub 仓库是否就是：https://github.com/12345-user/AIstorage.git
是否允许把本地 master 推送到 GitHub？
是否要提交当前所有配置改动？
```

完成后应达到：

- 本地分支有 upstream。
- GitHub 上能看到最新配置。
- 后续每次改知识库都可以提交和推送。

### 第 5 步：安装或确认剪藏工具

目标：网页和社媒内容能快速进入 Vault。

需要你选择一种：

```text

```

完成后应达到：

- 网页文章、公众号内容、视频字幕、灵感能按统一格式进入 Inbox。

### 第 6 步：接入 Horizon 或替代抓取工具

目标：自动抓 GitHub Trending、RSS、AI 工具新闻。

需要你提供：

```text

抓取关键词：

```

如果没有 Horizon，也可以先用替代方案：

- 手动 RSS 清单。
- GitHub Trending 手动剪藏。
- Codex 定期根据给定链接整理。

完成后应达到：

- 自动抓取结果进入 `00_Inbox/自动抓取`。
- Codex 可以定期生成每日简报。

### 第 7 步：配置文档、飞书、视频和配图

目标：补齐非网页资料处理能力。

按优先级做：

1. Word/旧文档：确认使用 Obsidian Importer、Docxer、Pandoc，或 Codex Documents。
2. 飞书：提供飞书导出文件或 API 授权。
3. 视频摘要：提供视频链接、字幕文件或转写工具。
4. 配图：确定默认尺寸、风格、命名规则。

建议先用这个规则：

```text
图片保存：99_Attachments/图片
PDF 保存：99_Attachments/PDF
表格保存：99_Attachments/表格
视频保存：99_Attachments/视频素材
```

### 第 8 步：跑一次端到端测试

目标：确认整个链路真的可用。

测试材料可以是一条网页链接或一段手动内容：

```text
标题：
链接 / 内容：
PROJECT：待判断
CMD：整理
```

Codex 应完成：

1. 补齐标准识别区。
2. 判断 PROJECT。
3. 判断 ACTION。
4. 写入正确目录。
5. 生成知识卡或选题。
6. 纳入每日简报。
