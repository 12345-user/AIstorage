# Codex 知识库工作规则

本仓库是一个 Obsidian 自动知识库。Codex 在处理资料、生成笔记或整理输出时，优先遵循这些规则。

## 默认目标

- 当前采用多项目结构，不再只有一个研究领域。
- 主要项目：客栈自媒体脚本与热点、生物信息研究生课程与实验报告、大学课程日常跟进、AI 前沿与 Codex 工具、其他杂事项目。
- 主要输出：短视频脚本、热点简报、实验报告、课程笔记、工具评测、文章、报告、周报。

## 文件夹职责

- `00_Inbox`：所有未处理信息入口，包括自动抓取和手动剪藏。
- `01_Sources`：原始资料，保留来源、作者、链接、日期和摘要。
- `02_Knowledge`：拆解后的知识卡片，一张卡片只表达一个核心观点或概念。
- `03_Topics`：选题池、研究问题、长期项目和专题路线。
- `04_Outputs`：可交付成品，包括文章、报告、脚本、课程资料和周报。
- `05_Review`：每日简报、每周复盘、月度总结和知识库优化记录。
- `06_Skills`：信息筛选、选题判断、资料总结、复盘和输出写作规则。
- `10_Projects`：五个主要项目的专属资料、进度和输出。
- `20_Shared`：跨项目复用资料，包括账号渠道、写作方法、AI 工具和客栈素材库。
- `99_Attachments`：图片、PDF、表格、视频等附件素材。

## 项目分流规则

新资料先判断属于哪个项目：

- P1 客栈自媒体脚本与热点：热点、对标账号、短视频脚本、客栈卖点、发布复盘。
- P2 生物信息研究生课程与实验报告：生物信息课程、实验记录、文献、实验报告。
- P3 大学课程日常跟进：日常课程资料、课堂笔记、作业、考试复习。
- P4 AI 前沿与 Codex 工具：AI 新闻、工具评测、Codex 实践、自动化流程。
- P5 其他杂事项目：暂时无法归入 P1-P4 的零散事务。

如果资料能服务两个以上项目，放入 `20_Shared`，并在相关项目首页或笔记中建立链接。

## 自动检查规则

处理新资料时，按以下五项判断：

1. 相关性：是否与研究领域、长期项目或输出目标有关。
2. 新鲜度：是否是新趋势、新案例、新工具或新数据。
3. 价值度：是否值得保存、引用、研究或复盘。
4. 可输出性：是否可以转化为文章、报告、脚本、课程或周报素材。
5. 关联性：是否能连接已有笔记、主题、资料或输出。

## 分类动作

- `P 进项目`：放入 `10_Projects` 对应项目。
- `X 进共享区`：放入 `20_Shared`。
- `S 进选题池`：放入 `03_Topics/选题池` 或项目内选题位置。
- `A 做知识卡`：整理到 `02_Knowledge`。
- `B 存参考资料`：归档到 `01_Sources`。
- `C 归档`：放到对应资料或输出目录，保留最小摘要。
- `D 忽略`：低相关、低价值、重复或无法验证的内容不入库。

## 输出要求

- 新笔记优先使用 `_templates` 中的模板。
- 每篇资料笔记必须包含来源、日期、摘要和下一步动作。
- 重要观点尽量拆成知识卡片，并链接到相关主题。
- 成品输出保存在 `04_Outputs`，复盘保存在 `05_Review`。
