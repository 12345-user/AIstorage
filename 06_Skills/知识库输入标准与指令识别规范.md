## 4. 重要性识别与去重规则

### 4.1 重要性字段

| 重要性 | 判断标准 | 默认处理 |
| --- | --- | --- |
| 高 | 含原始证据、真实数据、关键链接、项目决策、测试结果、截止日期、可交付成果、长期复用知识 | 入库并保留 |
| 中 | 阶段总结、资料清单、待办列表、局部复盘、对单一项目有用的信息 | 合并、归档或保留一份 |
| 低 | 重复摘要、同步副本、空壳索引、无新增判断的流水账、可从主文件再生成的草稿 | 删除、忽略或不入库 |

### 4.2 去重判定

如果两篇或多篇笔记内容重叠，按以下顺序保留：

1. 原始资料或最完整源文件。
2. 最终交付件。
3. 信息密度最高的报告、清单或知识卡。
4. 含明确下一步动作的版本。

删除或合并：

- 只指向另一篇文件、没有新增内容的同步副本。
- 已被报告、周报或简报覆盖的流程流水账。
- 标题不同但事实和结论完全相同的重复卡片。
- 被新版完全替代且无历史价值的旧稿。

### 4.3 删除保护

以下文件默认不删除，除非用户明确要求：

- 原始资料、来源笔记、附件索引。
- 已发布文章、脚本、报告、实验报告。
- 包含测试结果、配置要求、项目决策或真实数据的文件。
- 用户手动创建且无法确认用途的文件。

## 5. 指令识别格式

Codex 识别两种写法：自然语言写法和结构化写法。结构化写法优先级更高。

### 5.1 结构化写法

```text
CMD: 整理
PROJECT: P4
ACTION: A
OUTPUT: 知识卡
TARGET: 02_Knowledge/工具卡片
PRIORITY: 高
NOTE: 从这条资料提炼 2 张工具卡片，并关联 P4 项目首页。
```

### 5.2 单行写法

```text
CMD:整理 | PROJECT:P1 | ACTION:S+A | OUTPUT:视频脚本 | PRIORITY:高
```

### 5.3 一键全流程写法

用于需要从一条资料同时生成项目入库、知识卡、选题、脚本、文章、报告、周报和每日简报的场景。

```text
CMD:全流程 | FLOW:P1-HORIZON-FULLFLOW | PROJECT:P1 | ACTION:P+A+S+C | OUTPUT:知识卡+选题+视频脚本+文章+报告+周报+每日简报 | PRIORITY:高
```

`P1-HORIZON-FULLFLOW` 默认链路：

```text
整理 -> 入库 -> 知识卡 -> 选题 -> 脚本 -> 文章 -> 报告 -> 周报 -> 简报
```

项目复盘为可选产物：只有存在发布数据、决策变化或明确经验时才生成。

默认目标：

- 项目入库：`10_Projects/01_客栈自媒体脚本与热点/00_项目收件箱`
- 知识卡：`02_Knowledge`
- 选题：`03_Topics/选题池`
- 脚本：`04_Outputs/视频脚本`
- 文章：`04_Outputs/文章`
- 报告：`04_Outputs/报告`
- 周报：`04_Outputs/周报总结`
- 简报：`05_Review/每日简报`
- 复盘：可选，`10_Projects/01_客栈自媒体脚本与热点/05_发布复盘`

### 5.4 自然语言写法

```text
请把这条资料按 P4 处理，先存参考资料，再做 1 张知识卡，并纳入今天每日简报。
```

## 6. 指令关键词映射

| 用户说法 | Codex 识别为 | 默认动作 |
| --- | --- | --- |
| 整理 / 处理 / 分类 | `CMD: 整理` | 做五项检查并给出去向 |
| 入库 / 保存 | `CMD: 入库` | 根据分类动作写入目标目录 |
| 做知识卡 / 提炼观点 | `CMD: 知识卡` | `ACTION:A` |
| 生成选题 / 放进选题池 | `CMD: 选题` | `ACTION:S` |
| 写脚本 / 短视频 | `CMD: 脚本` | 保存到 `04_Outputs/视频脚本` |
| 写文章 / 成文 | `CMD: 文章` | 保存到 `04_Outputs/文章` |
| 写报告 / 实验报告 | `CMD: 报告` | 保存到 `04_Outputs/报告` 或 P2 报告目录 |
| 生成周报 | `CMD: 周报` | 保存到 `04_Outputs/周报总结` |
| 生成简报 / 今日简报 | `CMD: 简报` | 保存到 `05_Review/每日简报` |
| 全流程 / 一键全流程 / 跑完整链路 | `CMD: 全流程` | 根据 `FLOW` 执行多产物输出 |
| 复盘 | `CMD: 复盘` | 保存到 `05_Review` |
| 归档 | `CMD: 归档` | `ACTION:C` |
| 忽略 / 不要保存 | `CMD: 忽略` | `ACTION:D` |
| 删除低重要性 / 去重清理 | `CMD: 清理` | 按重要性和去重规则删除或合并 |

## 7. Codex 执行顺序

1. 读取“标准识别区”。
2. 若字段缺失，按内容推断；无法推断则写“待补”。
3. 优先执行 `CMD`、`PROJECT`、`ACTION`、`TARGET` 明确指定的内容。
4. 未指定时，按五项检查自动给出项目归属和分类动作。
5. 新建笔记必须使用 `_templates` 中含有“标准识别区”的模板。
6. 所有输出必须写明保存路径、关联笔记和下一步动作。
7. 清理文件时先判断重要性；低重要性且重复的文件可删除，中重要性文件优先合并，高重要性文件默认保留。

```text

CMD：整理 / 入库 / 知识卡 / 选题 / 脚本 / 文章 / 报告 / 周报 / 简报
```
