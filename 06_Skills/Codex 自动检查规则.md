# Codex 自动检查规则

本规则必须与 `06_Skills/知识库输入标准与指令识别规范.md` 保持一致。

## 先读标准识别区

Codex 处理资料时，先读取以下字段：

- `CMD` / `处理指令`
- `PROJECT` / `项目归属`
- `ACTION` / `分类动作`
- `OUTPUT` / `目标输出`
- `TARGET` / `保存路径`
- `PRIORITY` / `优先级`

结构化指令优先于正文推断。字段缺失时，按内容推断；无法推断则写“待补”。

## 项目归属判断

- P1 客栈自媒体脚本与热点
- P2 生物信息研究生课程与实验报告
- P3 大学课程日常跟进
- P4 AI 前沿与 Codex 工具
- P5 其他杂事项目
- X 共享资料

## 五项判断

1. 相关性：是否服务于研究领域、长期项目或当前输出。
2. 新鲜度：是否包含新趋势、新案例、新工具或新数据。
3. 价值度：是否值得保存、引用、研究或复盘。
4. 可输出性：是否能变成文章、报告、脚本、课程或周报。
5. 关联性：是否能连接已有笔记、主题、资料或输出。

## 分类动作

| 动作 | 含义 | 保存位置 |
| --- | --- | --- |
| P | 进项目 | `10_Projects` 对应项目 |
| X | 进共享区 | `20_Shared` |
| S | 进选题池 | `03_Topics/选题池` 或项目内选题位置 |
| A | 做知识卡 | `02_Knowledge` |
| B | 存参考资料 | `01_Sources` |
| C | 归档 | 对应资料目录或输出目录 |
| D | 忽略 | 不入库 |

## 输出格式

每次整理资料时，必须给出：

- 标准识别区是否完整。
- 分类结果：`PROJECT` 和 `ACTION`。
- 推荐保存路径：`TARGET` 或推断路径。
- 一句话摘要。
- 可关联的已有笔记。
- 下一步动作。
